* cl-llama-cpp

Common Lisp bindings to [[https://github.com/ggml-org/llama.cpp][llama.cpp]] via CFFI.

The generated =%llama= layer covers the complete public C API (881 functions, 38 enums,
59 structs). A higher-level =cl-llama-cpp= package wraps common workflows — text
generation, chat, embeddings, constrained generation, and more — with idiomatic Lisp
macros, typed handles, and structured conditions. =%llama= is a first-class escape
hatch, not an implementation detail: when a high-level wrapper doesn't exist, call the
C function directly. A missing wrapper is never a blocker.

*Active Development / Beta.* API is unstable and will change without notice.

** Requirements

- *SBCL* (recommended) — llama.cpp triggers floating-point exceptions that SBCL masks
  automatically. Other implementations are supported via a configurable wrapper.
- [[https://github.com/ggml-org/llama.cpp][llama.cpp]] built as a shared library (=libllama.so= / =libllama.dylib=)
- [[https://cffi.common-lisp.dev/][CFFI]] and [[https://github.com/cffi/cffi][cffi-libffi]]
- [[https://github.com/trivial-garbage/trivial-garbage][trivial-garbage]]

Tested on Linux. macOS should work (=.dylib= is handled). Windows is untested.

** Installation

The repository includes llama.cpp as a git submodule:

#+begin_src bash
git clone --recursive https://github.com/licjon/cl-llama-cpp
cd cl-llama-cpp/llama.cpp
cmake -B build -DBUILD_SHARED_LIBS=ON
cmake --build build -j$(nproc)
#+end_src

The library loader finds =libllama.so= in =llama.cpp/build/bin/= automatically.
Alternatively, install it somewhere on your system library path.

Place or symlink this directory into your ASDF source registry (e.g. =~/common-lisp/=):

#+begin_src lisp
(asdf:load-system "cl-llama-cpp")
#+end_src

Roswell users: clone into =~/.roswell/local-projects/= and it is found automatically.
Quicklisp users: run =ql:register-local-projects= first.

** Quick Start

#+begin_src lisp
(cl-llama-cpp:with-model (model "/path/to/model.gguf" :n-gpu-layers 99)
  (cl-llama-cpp:with-context (ctx model :n-ctx 2048)
    (princ (cl-llama-cpp:generate ctx "The capital of France is"
                                  :max-tokens 64 :temp 0.7))))
#+end_src

Streaming:

#+begin_src lisp
(cl-llama-cpp:with-model (model "/path/to/model.gguf" :n-gpu-layers 99)
  (cl-llama-cpp:with-context (ctx model :n-ctx 2048)
    (cl-llama-cpp:generate ctx "Once upon a time"
                           :max-tokens 128 :temp 0.9
                           :token-callback (lambda (tok)
                                             (write-string tok)
                                             (force-output)
                                             t))))
#+end_src

Multi-turn chat with KV-cache reuse — each turn decodes only the new tokens:

#+begin_src lisp
(cl-llama-cpp:with-model (model "/path/to/chat-model.gguf" :n-gpu-layers 99)
  (cl-llama-cpp:with-context (ctx model :n-ctx 4096)
    (let ((session (cl-llama-cpp:make-chat-session ctx
                     :system-prompt "You are a helpful assistant.")))
      (format t "~A~%"
              (cl-llama-cpp:chat-session-send session "What is Common Lisp?"
                                              :max-tokens 256))
      (format t "~A~%"
              (cl-llama-cpp:chat-session-send session "Tell me more about CLOS."
                                              :max-tokens 256)))))
#+end_src

** High-Level API

- *Text generation* — =generate= accepts 15 sampling strategies (temperature, top-k/p,
  min-p, mirostat v1/v2, XTC, DRY, typical-p, top-n-sigma, dynamic temperature,
  repetition/frequency/presence penalties, logit bias, greedy). Reusable sampler config
  objects (=make-sampler-config=) and explicit sampler chains (=with-sampler-chain=,
  individual =make-*-sampler= constructors).
- *Chat* — Simple sessions (full-history prefill) and incremental sessions
  (=make-chat-session= / =chat-session-send=: KV-cache reuse keeps per-turn prefill
  cost proportional to new tokens only). Chat template formatting and safe multi-turn
  tokenization (=format-chat=, =tokenize-chat=).
- *Constrained generation* — GBNF grammars, lazy grammars with trigger words, infill.
- *Parallel decoding* — =generate-parallel= runs multiple prompts simultaneously in
  shared forward passes. Independent contexts can run concurrently on separate threads.
- *Embeddings* — =embed= with optional normalization; context created with =:embeddings t=.
- *LoRA adapters* — =with-lora= / =apply-lora= with scale control.
- *KV cache* — clear, copy, shift, divide, per-sequence position queries.
- *Session state* — save/load to disk (survives process restarts) or to in-memory octet
  vectors (context forking, snapshotting).
- *GGUF inspection* — =with-gguf= reads file-level metadata, typed KV entries (including
  arrays), and tensor info without loading weights or initializing the backend.
- *Resource planning* — =estimate-memory=, =validate-configuration=,
  =suggest-configuration=, =feasibility-report= before committing to a model load.
  Optional pre-creation guardrails in =with-context= (=:validation :warn/:error=).
- *Introspection* — model description and metadata, architecture properties, context
  config, backend device enumeration, system capabilities.
- *Performance* — =with-perf= macro, structured timing data, sampler timing.
- *Logging* — route llama.cpp log output to a Lisp callback (=set-log-callback=).
- *Backend lifecycle* — =with-backend=, =ensure-backend=, NUMA init, threadpool
  management, abort callbacks.

Conditions inherit from =llama-error= with structured slots (e.g. =model-load-error=
carries =path=, =decode-error= carries =code=, =input-validation-error= carries
function/argument/value/reason). Resource-loading conditions also establish interactive
restarts for the debugger: retry with different params, use CPU only, skip LoRA, etc.

Not yet wrapped at the high level (use =%llama= directly):

- Model quantization

** Low-Level =%llama= Access

=%llama= is a supported, first-class layer — not an implementation detail. All 881
public C functions are available. The naming convention strips the =llama_= prefix and
converts underscores to hyphens (=llama_decode= → =%llama:decode=,
=LLAMA_SPLIT_MODE= → =%llama:split-mode=); =ggml_= and =gguf_= prefixes are kept.
Extract raw pointers from high-level handles with =llama-model-pointer=,
=llama-context-pointer=, etc.

Direct =%llama= calls must run inside =with-llama-compatible-fp-environment= to mask
SBCL's FP traps. The high-level API handles this automatically.

See [[docs/low-level.org]] for naming conventions, struct and enum usage, and worked
examples (including model quantization).

** Running Tests

Smoke tests (no model required):

#+begin_src bash
sbcl --eval '(ql:quickload "cl-llama-cpp/tests")' \
     --eval '(asdf:test-system "cl-llama-cpp/tests")' \
     --eval '(quit)'
#+end_src

Integration tests require GGUF model files set via environment variables
(=LLAMA_TEST_MODEL=, =LLAMA_TEST_EMBED_MODEL=, =LLAMA_TEST_LORA=):

#+begin_src bash
LLAMA_TEST_MODEL=/path/to/model.gguf \
LLAMA_TEST_EMBED_MODEL=/path/to/embedding-model.gguf \
LLAMA_TEST_LORA=/path/to/adapter.gguf \
sbcl --eval '(ql:quickload "cl-llama-cpp/tests")' \
     --eval '(asdf:test-system "cl-llama-cpp/tests")' \
     --eval '(quit)'
#+end_src
